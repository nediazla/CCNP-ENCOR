# **Capítulo 22 – Enterprise Network Architecture**

La arquitectura de red empresarial moderna combina elementos clásicos (Core/Distribution/Access) con nuevos paradigmas como **SDN, virtualización, segmentation, overlay networks y control automatizado**.

Este capítulo analiza los modelos Cisco **Campus LAN**, **WAN**, **Data Center**, **Branch**, y cómo se integran en el concepto general de red empresarial.

---

# **1. Dominio Enterprise Network Architecture**

Cisco divide la red empresarial en **cinco dominios principales**:

1. **Enterprise Campus**
    
    - LAN cableada
        
    - LAN inalámbrica (WLC, APs)
        
    - Seguridad local
        
    - Servicios de capa 2/3
        
2. **Enterprise WAN**
    
    - MPLS
        
    - SD-WAN
        
    - VPN site-to-site
        
    - Internet empresarial
        
3. **Small/Remote Branch**
    
    - Routers ISR
        
    - APs remotos
        
    - Controladores en nube (Meraki, Catalyst Wireless)
        
4. **Enterprise Data Center**
    
    - Switching L3
        
    - Fabric VXLAN/EVPN
        
    - UCS, virtualización, servidores
        
5. **Cloud / Multicloud**
    
    - AWS / Azure / GCP
        
    - Cloud Edge
        
    - SD-WAN fabric a la nube
        

Cada dominio tiene funciones distintas, pero deben integrarse como una red coherente.

# **2. Modelos clásicos de diseño de red**

Cisco ha usado históricamente un modelo de tres capas:

## **2.1. Core Layer**

- Alta velocidad
    
- Alta disponibilidad
    
- Enrutamiento entre edificios
    
- Sin ACLs o políticas pesadas (para evitar cuello de botella)
    

## **2.2. Distribution Layer**

- Control de políticas
    
- ACLs
    
- Redundancia
    
- Agregación de enlaces
    
- Límite entre L2 y L3
    

## **2.3. Access Layer**

- Conectividad de usuarios
    
- PoE para teléfonos y APs
    
- VLANs
    
- Autenticación (802.1X)
    
- QoS para voz/video
    

### Otros diseños:

- **Two-tier (Collapsed Core)**  
    Core + Distribution fusionados (típico en redes pequeñas)

# **3. Arquitectura Enterprise moderna: SDN y Automation**

El diseño moderno se basa en:

- **Separation of control/data plane**
    
- **Programmability** (APIs, YANG models)
    
- **Automation** (NETCONF/RESTCONF/Ansible)
    
- **Network Assurance** (telemetría continua)
    
- **Intent-Based Networking (IBN)**
    

El objetivo es transformar la red en algo **más predecible y gestionable**, reduciendo el trabajo manual.

---

# **4. Control Plane, Data Plane y Management Plane**

### **Control Plane**

- Construye rutas
    
- Protocolos como OSPF, EIGRP, BGP
    
- En switches: STP, RSTP, MST
    
- En SDN: control centralizado (Catalyst DNA Center, SD-Access)
    

### **Data Plane**

- Reenvía paquetes
    
- CEF (Cisco Express Forwarding)
    
- ACLs, QoS, NAT en hardware
    

### **Management Plane**

- SNMP
    
- Syslog
    
- Telemetría
    
- NETCONF/RESTCONF
    
- SSH
    

La separación clara permite automatización y troubleshooting más fácil.

# **6. Diseño de Enterprise WAN**

Tecnologías actuales:

### **MPLS L3VPN**

- Transporte privado
    
- QoS garantizado
    
- Integración con BGP
    

### **SD-WAN (Cisco vManage / IOS-XE SD-WAN)**

- Controlador central
    
- Automatización de políticas
    
- Path selection basado en SLA
    
- Integración a multicloud
    
- Overlays con IPsec
    

### **Internet Dual / Hybrid WAN**

- MPLS + Internet
    
- Internet dual active/active
    
- Local breakout
    

---

# **7. Virtualización en redes Enterprise**

### **VRF-Lite**

- Múltiples tablas de routing
    
- Separación lógica de tráfico
    
- Muy usado en campus
    

### **MPLS dentro de proveedores**

- Transporte VPN
    
- Segmentación completa
    

### **Overlay VXLAN/EVPN**

- Usado en Data Center
    
- Transporte L2 sobre L3
    
- 16 millones de VNIs
    
- Control plane mediante BGP EVPN
    

---

# **8. Segmentación empresarial**

Fundamental para seguridad y control.

### **Macro-segmentation**

- División por VRFs / VLANs
    
- Perímetro (firewalls)
    

### **Micro-segmentation**

- Identity-based policies
    
- Cisco TrustSec (SGTs)
    
- SD-Access Scalable Groups
    

Esto permite controlar tráfico intra-corporativo de forma granular.

---

# **9. Cisco SD-Access (Campus Fabric)**

Modelo moderno para redes Campus.

### **Componentes**

- **Fabric Edge Nodes**
    
- **Fabric Border Nodes**
    
- **Control Plane Nodes (LISP)**
    
- **Wireless Integration (Fabric APs)**
    
- **DNA Center** como controlador
    

### **Protocolos clave**

- VXLAN para overlay
    
- LISP para control-plane (host tracking)
    
- Cisco TrustSec para políticas
    

### **Beneficios**

- Movilidad de usuarios
    
- Segmentación automática
    
- Políticas centralizadas
    
- Integración cableado + inalámbrico

# **10. Arquitectura de Data Center (visión general)**

Aunque el examen no profundiza tanto como CCNP DC, debes conocer:

### **Leaf-Spine**

- Spine switches = backbone
    
- Leaf switches = borde
    
- Todos los leaf se conectan a todos los spine
    
- Latencia uniforme y escalabilidad
    

### **Fabric VXLAN/EVPN**

- Reemplaza STP
    
- BGP EVPN como control-plane
    
- VXLAN como overlay

# **12. High Availability (HA)**

Presente en todos los dominios enterprise:

- HSRP/VRRP/GLBP
    
- MEC / EtherChannel
    
- ECMP (Equal Cost Multipath)
    
- Controladoras en HA (N+1, SSO)
    
- SD-WAN con enlaces redundantes

- Modelos YANG

- Baselines de rendimiento
